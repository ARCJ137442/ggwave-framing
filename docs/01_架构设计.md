# GGWave 分包算法项目 - 架构设计

> 项目：ggwave-framing | 版本：v0.1.0 | 更新：2026-05-07

---

## 一、项目定位

### 1.1 核心目标

**不修改 ggwave 库本身**，在其之上封装分层协议，实现任意长度数据的可靠分包传输。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **无侵入** | 不修改 ggwave-rs，通过接口组合实现 |
| **单向传输** | 适用场景：WAV 文件、广播、录音场景 |
| **简单优先** | KISS 原则，不过度设计 |
| **可测试** | 每个模块可独立测试验证 |

## 三、模块设计

### 3.1 模块列表

| 模块 | 职责 | 对外接口 |
|------|------|---------|
| `framer` | 分片/重组核心逻辑 | `Fragment::new()`, `add_frame()`, `is_complete()` |
| `protocol` | 帧头编码/解码 | `encode_header()`, `decode_header()` |
| `codec` | GGWave 封装 | `encode_frame()`, `decode_frame()` |
| `wav` | WAV 文件读写 | `write_wav()`, `read_wav()` |
| `error` | 错误类型定义 | `FramingError` 枚举 |

### 3.2 接口设计

#### Framer 模块

```rust
pub struct Framer {
    pub total_frames: u16,      // 总帧数
    pub max_payload: usize,      // 每帧最大载荷
    seq: Vec<Option<Vec<u8>>>, // 按序号存储的帧数据
}

impl Framer {
    /// 创建发送端 Framer
    pub fn new(total_frames: u16, max_payload: usize) -> Self;

    /// 获取指定序号帧的完整载荷（含帧头）
    pub fn get_frame(&self, seq: u16) -> Vec<u8>;

    /// 创建接收端 Framer
    pub fn receiver(total_frames: u16, max_payload: usize) -> Self;

    /// 添加接收到的帧
    pub fn add_frame(&mut self, seq: u16, payload: &[u8]) -> Result<(), FramingError>;

    /// 检查是否收齐
    pub fn is_complete(&self) -> bool;

    /// 提取完整数据
    pub fn extract(&self) -> Result<Vec<u8>, FramingError>;
}
```

#### Protocol 模块

```rust
/// 帧头结构
#[repr(C)]
pub struct FrameHeader {
    pub version: u8,      // 协议版本，0x01
    pub frame_type: u8,   // 帧类型：DATA=0x01, EOF=0x02, ACK=0x03
    pub seq: u16,         // 序号，小端序
    pub total: u16,      // 总帧数，小端序
}

impl FrameHeader {
    /// 从字节 slice 解析帧头
    pub fn decode(data: &[u8]) -> Result<Self, FramingError>;

    /// 编码为字节 Vec
    pub fn encode(&self) -> Vec<u8>;

    /// 获取帧类型
    pub fn frame_type(&self) -> FrameType;
}

/// 帧类型枚举
#[derive(Debug, Clone, Copy)]
pub enum FrameType {
    Data,  // 数据帧
    Eof,   // 结束帧
    Ack,   // 确认帧（预留）
}
```

#### Codec 模块（GGWave 封装）

```rust
pub struct GGWaveCodec {
    gg: GGWave,
    protocol: ProtocolId,  // 使用 AUDIBLE_FAST
}

impl GGWaveCodec {
    /// 编码单帧数据为音频
    pub fn encode_frame(&self, data: &[u8]) -> Result<Vec<f32>, FramingError>;

    /// 解码音频为原始数据
    pub fn decode_frame(&self, audio: &[f32]) -> Result<Vec<u8>, FramingError>;

    /// 从 WAV 数据解码
    pub fn decode_from_wav(&self, wav_data: &[u8]) -> Result<Vec<u8>, FramingError>;
}
```

### 3.3 数据流

#### 编码流程

```
应用数据 (bytes)
    │
    ▼ Framer::new(total_frames, max_payload)
    │
    ▼ for each frame:
    │   frame.header = FrameHeader { version, type, seq, total }
    │   frame.payload = data[seq*max_payload .. (seq+1)*max_payload]
    │   frame.full = header + payload
    │
    ▼ Codec::encode_frame(frame.full)
    │
    WAV 音频数据
```

#### 解码流程

```
WAV 文件
    │
    ▼ Codec::decode_from_wav()
    │
    ▼ Protocol::decode_header(frame)
    │
    ▼ Framer::receiver(total)
    │
    ▼ Framer::add_frame(seq, payload)
    │
    ▼ Framer::is_complete()
    │
    ▼ Framer::extract()
    │
    应用数据 (bytes)
```

## 四、帧头设计

### 4.1 帧头格式

```
Byte 0: [版本 4bits | 类型 4bits]
Byte 1-2: 序号 (u16, little-endian)
Byte 3-4: 总帧数 (u16, little-endian)
```

| 字段 | 位 | 说明 |
|------|---|------|
| version | 4 | 协议版本，固定 0x1 |
| type | 4 | 帧类型：0x1=Data, 0x2=Eof |
| seq | 16 | 帧序号 (0 ~ total-1) |
| total | 16 | 总帧数 |

**帧头大小：5 字节**

### 4.2 帧类型定义

| type 值 | 名称 | 说明 |
|---------|------|------|
| 0x1 | DATA | 数据帧 |
| 0x2 | EOF | 结束帧（最后一帧） |

### 4.3 版本协商

未来版本扩展：
- version 0x1: 当前版本
- version 0x2: 支持 ACK 确认
- version 0x3+: 预留

## 五、参数设计

### 5.1 核心参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 帧头大小 | 5 bytes | version+type+seq+total |
| 最大载荷 | 95 bytes | 140 - 5 - 40 ECC余量 |
| 帧总大小 | ~100 bytes | 帧头 + 载荷 + GGWave开销 |
| 协议选择 | AUDIBLE_FAST | 平衡速度(24B/s)和可靠性 |

### 5.2 传输计算

横纵分析报告（20,123 bytes）：

```
总帧数 = ceil(20123 / 95) = 212 帧

⚠️ 注意：GGWave 是实时协议，每帧音频 2-16 秒（与理论字节速率完全不同）

实际测试结果：
  AUDIBLE_FASTEST: 每帧 ~2.4s → 总计约 500s (~8.5分钟)
  AUDIBLE_FAST:    每帧 ~4.1s → 总计约 870s (~14.5分钟)
  AUDIBLE_NORMAL:  每帧 ~5.9s → 总计约 1250s (~21分钟)

理论字节速率估算（24 B/s）：212 × (100/24) ≈ 883ms — 与实际严重不符！
```

## 七、错误处理

### 7.1 错误类型

```rust
#[derive(Debug, Clone)]
pub enum FramingError {
    // 协议错误
    InvalidVersion(u8),           // 无效版本号
    InvalidFrameType(u8),         // 无效帧类型
    InvalidSeq { expect: u16, got: u16 },  // 序号不匹配
    InvalidTotal { first: u16, conflict: u16 },  // 总帧数不一致

    // 重组错误
    IncompleteFrames { received: u16, total: u16 },  // 帧不完整
    EmptyPayload,               // 空载荷
    BufferTooSmall,             // 缓冲不足

    // IO 错误
    IoError(String),            // 文件读写错误

    // GGWave 错误
    GGWaveError(String),        // GGWave 编码/解码错误
}
```

### 7.2 错误处理策略

| 错误类型 | 处理策略 | 说明 |
|---------|---------|------|
| InvalidVersion | 记录并跳过 | 版本不兼容，跳过该帧 |
| InvalidSeq | 记录并跳过 | 丢包或乱序，跳过 |
| InvalidTotal | 记录并使用第一个 | 取第一个帧的 total |
| IncompleteFrames | 返回错误 | 需要所有帧 |
| IoError | 传播错误 | 上层处理 |
